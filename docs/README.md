# Documentation Directory

This directory contains all project documentation organized by category.

## Directory Structure

### `/deployment/` - Deployment Documentation
Documentation related to deploying the application and its components:
- Deployment checklists
- Deployment status reports
- Deployment guides and summaries

### `/setup/` - Setup and Configuration
Documentation for initial setup and configuration:
- Setup checklists
- Email configuration guides
- Copilot and development environment setup

### `/features/` - Feature-Specific Documentation
Documentation for specific features and their implementation:
- Face recognition and training
- Auto-training implementation
- Database improvements and fixes
- Feature summaries

### `/development/` - Development Documentation
Documentation for developers working on the project:
- Change summaries
- Pull request details
- Development notes

### Root Level Documentation
Core documentation files in the root `/docs/` directory:
- `AZURE_SETUP.md` - Azure resources configuration
- `AUTHENTICATION_SETUP.md` - OAuth setup instructions
- `RBAC_SYSTEM.md` - Role-based access control documentation
- `FACE_RECOGNITION.md` - Face recognition system overview
- And other technical guides

## Main Project Documentation

Key documentation in the project root:
- `/README.md` - Project overview and quick start
- `/RBAC_DEPLOYMENT_CHECKLIST.md` - RBAC deployment guide

## Navigation Tips

- Start with the main `/README.md` for project overview
- Use setup guides in `/docs/setup/` for initial configuration
- Refer to `/docs/deployment/` when deploying changes
- Check `/docs/features/` for specific feature documentation
- Review `/docs/development/` for development-related information
